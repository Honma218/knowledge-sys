### チャプター 1: オープニング（0:07〜）

京平さんが3年前ぐらいに書いた「GitHubで人生を管理する」記事が、最近またXで話題になった。書いたのは2021年ごろ。プロジェクト管理ツール（タスク管理ツール）を仕事だけで使うのではなく、家族のこと、歯医者に行くこと、株を始めることまで、何でもGitHubで管理したら面白いのでは、という発想から始まった。人生のタスクやイベントを全部管理する感じ。

### チャプター 2: 概要（0:54〜）

「life」というリポジトリをかなり長く運用している。4年前から運用していて、一時期 Notion に移行していたが、またGitHubに戻ってきた。Notion からの脱却にも成功している。

移行方法：Notion のカンバンビューから CSV をエクスポート → その CSV を Claude Code に渡す → Claude Code は `gh` コマンド（GitHub CLI）を使えるので、Issue を 36個 一気に作成してくれた。

### チャプター 3: lifeリポジトリの使い方（3:35〜）

（オーナーシップ・プライバシー関連の運用上の課題について言及）OSS のプロジェクトをやっていると Issue がどんどん立つが、それを放置しないように取り崩す作業がとても大事。この考え方を life リポジトリでも重視している。

### チャプター 4: なぜ人生をGitHubで管理しようと思ったか（7:20〜）

- タスク管理ツールを作っている会社で働いていた経験がある
- GitHub が大好き、タスク管理について調べるのが好き
- 新卒のころ「タスクオンテーブル」（Excelライクにタスクを管理できる）アプリを作ったことがある
- 色々考えた結果「人生をタスク管理するのが最適解ではないか」という仮説に至った
- 「ルートルートだから、人生が」＝あらゆるプロジェクトの最上位に人生がある、の意

### チャプター 5: メリット・デメリット（8:20〜）

**メリット:**
- コンテキストスイッチングが要らない（仕事と人生を別々のツールで管理する必要がない）
- 仕事のリポジトリばかり見ている人は他プロジェクトのレポを見なくなる。全部同じレイヤーに乗せる

**デメリット:**
- チームプレイの観点ではまだ考える必要がある
- 「スコープを分けて共有できるのがベスト」— タグやラベルで「これはあの人に見せてもいいよね」の制御を目指したい
- セキュリティレベルを示すタグで秘密情報を分ける、というアイデア

### チャプター 6: なぜNotionから移行したか（11:00〜）

- **Notion は AI が使いづらい**（お金を払って使いたいとまでは思わなかった）
- Claude Code がそのまま使えるメリットが大きい
- Claude の Skills（スキル）の作り方を覚えたら、開発にも life リポジトリにも活かせる
- **使うツールをどんどん小さく／狭くしていきたい**（断捨離）
- 新しいツールの昨日のリリースなどにキャッチアップする余裕がない

### チャプター 7: どういう情報を管理しているか（12:00〜）

主要ディレクトリ構造：

- **Portfolio** — リポジトリ内のサブディレクトリに「今までどういう仕事をしたか」「登壇履歴」などを全部載せる。問い合わせが来たとき「このリポジトリが誰よりも自分のことをよく知っている」状態にする
- **技術書**（ディレクトリ） — どういう本を作ろうとしているかの台本、参考資料
- **Ideas / Weekly** — 1週間で呟いた X の投稿をアグリゲーション、noteで記事にしたりメディア発信に使う
- **Publications** — ホームページや他メディアに発信したものの管理
- **YouTube** — 過去の YouTube 発言をすべて取り込み（Claude Code に Gemini を使わせて全取得）

この「メディアミックス」（ポスト→記事→動画、動画→記事 の相互変換）を最初からやりたかった。最近出した YouTube 動画は「リクス」（＝「リファクタリングス」？）についての X つぶやきからコンバートしたもの。

### チャプター 8: 実際の情報収集のワークフロー（14:30〜）

- 普通の開発では推奨しないが、自分の場合は脆弱性がない（個人情報がバレるだけ）ので Claude Code の「Dangerous Permissions」モードで動かす
- `/ネタトレンド-デイリー` というClaude Skill を実行 → 今日起きた事件やネタ（ネタ=記事ネタ）をトレンド形式で出してくれる

### チャプター 9: Claude Skills の活用（17:00〜）

- 自分のホームページのページ削除を Claude Code に依頼すると、適切なファイルを見つけて消してくれる
- **X の Weekly 集約**: 1週間の自分の投稿を CSV でダウンロード → インプレッション高いものを10個まとめる → `Ideas/Weekly` に配置 → 次のアクション（note記事、YouTube動画）につなげる
- 壁打ちを Agent Memory に入れる → 普段の Ideas/Daily のネタから X で呟く → 反応が来る → いい議論やトピックをまとめていく → Publications / Portfolio → ホームページや他メディアに発散
- 「人生の流れ」が作れている状態

### チャプター 10: データ管理・情報収集の自動化設定（19:05〜）

- フロントのネタをコーヒーを飲みながら見る運用
- Claude Skill のテンプレート（CLAUDE.md 相当）を慎重にチューニング
- **CLAUDE.md は 300行以下に抑えるべき**
  > これ入れれば入れるほどバカになるっていうか、平坦になる。発散、うまくコンテキスト読み込めないみたいな。
- この設計を組み上げるのに2ヶ月費やした
- 「誰かの方法を真似して参考にするのはいいけど、多分自分の用途に合わないから色々変えた方がいい」

### チャプター 11: 実際に使い始めて変わったこと（21:05〜）

- Notionを抜けられたこと
- 育休中、子育てしながら空いた時間で働く状況で、この仕組みがないと発信活動・セキュリティ勉強・本執筆準備・登壇準備がほぼできなかっただろう
- 情報収集 → 興味あるもの選定 → 書けるもの執筆、という流れ

### チャプター 14: issueを発行する基準・粒度（24:45〜）

- **山田しさん開発の「エージェントメモリー」** に聞く
  > 今まで貯めている間で一周できそうなのあるとか聞くと何か出してくれる
